## 这是什么？

一个 **Claude Code Skill** — 就是一个 Markdown 文件，能让 Claude Code 扮演一个完整的开发团队。不需要外部工具、不需要 Python 运行时、不需要 LangGraph。Claude Code 读取 Skill 定义后，在 PM（产品经理）、TechLead（技术负责人）、QA（测试工程师）三个专业角色之间轮换，驱动完整的软件开发生命周期。

**痛点：** 一个人写代码，容易漏需求、跳设计、代码风格不一致、上线前不测试。你需要一个团队，但你没有。

**解决：** 这个 Skill 赋予 Claude Code 三个专业角色和一个严格的 7 阶段工作流。新项目从零搭建、已有项目加功能改代码，都走同一套流程。它像虚拟开发团队一样强制结构化推进，而你在每个阶段转换时都拥有确认权。

## 为什么用它？

| 不用 Skill | 用了 Skill |
|---|---|
| Claude 上来就写代码 | PM 先澄清需求 |
| 没有架构规划 | TechLead 输出 Mermaid 架构图 + 技术选型对比 |
| 前端千篇一律 | 每个 UI 模块附带设计决策说明 |
| 前后端各写各的 | 先协商接口契约再动手写代码 |
| 没有系统测试 | QA 输出结构化测试报告，Bug 分级 |
| 代码裸奔交付 | 交付经理打包 README、Docker、CI/CD、Changelog |

## 三个角色

### PM（产品经理）
- 通过结构化问答澄清需求
- 输出包含用户故事和验收标准的完整 PRD
- 绝不讨论技术实现方式
- 每轮只问 1-3 个关键问题，不让人窒息

### TechLead（技术负责人，含 4 个子身份）
- **架构师** — 技术选型评估、Mermaid 架构图、风险分析
- **后端工程师** — API 设计、数据模型、业务逻辑、认证授权、中间件
- **前端工程师** — UI 组件、页面、状态管理，强制输出设计决策说明
- **交付经理** — 打包一切：Docker、CI/CD、部署脚本、Changelog

### QA（质量工程师）
- 静态代码审查（安全、规范、可维护性）
- 生成补充测试用例
- 输出结构化测试报告，Bug 分级
- Bug 修复循环上限 3 轮，防止无限往复

## 内置设计系统

每个前端模块都有设计决策说明 — 不会产出千篇一律的 UI。

| 类别 | 默认配置 |
|------|----------|
| **参考风格** | Linear / Vercel — 极简、专业、暗色模式 |
| **颜色** | 完整的深色 + 浅色 CSS 变量集，主色调可配置 |
| **字体** | Inter，8 级字号系统（xs → 4xl） |
| **间距** | 4px 基准，11 级间距 |
| **圆角** | 4 级：sm(按钮) / md(卡片) / lg(弹窗) / xl(面板) |
| **阴影** | 3 级，克制 — 暗色模式微不可察，亮色模式更明显 |
| **动画** | 150ms 微交互、列表交错入场、绝不做纯装饰性动画 |
| **组件** | shadcn/ui（Radix 基元）+ lucide-react 图标 |
| **状态** | 每个组件覆盖：骨架屏加载态、空状态引导、错误状态含重试按钮 |

随时可以覆盖风格："参考 Stripe 风格"、"用蓝色主色调"、"亮色模式"、"更活泼一点"。

## API 契约协商

防止前后端脱节的核心机制：

```
[后端工程师]
"这是我的 API 契约草案 v1.0：

GET  /api/users     → { data: User[], total, page }
POST /api/users     → { data: User }
GET  /api/users/:id → { data: User }

前端，这些接口满足需求吗？"

[前端工程师]
"审阅完毕，两点调整：
1. GET /api/users — User 里需要加 avatar_url 字段
2. 所有列表接口统一分页格式为 { items, total, page, pageSize }

除此以外没问题。确认契约 v1.1。"

[后端工程师]
"契约 v1.1 已确认。按此开发。"
```

双方按约定的接口各自开发，联调时零意外。

## 内置安全规则

OWASP Top 10 规则融入代码生成。QA 将违规视为阻断级 Bug：

- 禁止 `eval()`、`exec()`、`os.system()` 接受用户输入
- 禁止硬编码密钥 — 一律用 `process.env`
- 强制参数化查询 — 禁止 SQL 字符串拼接
- 所有 API 入口校验和清洗用户输入
- 禁止不加清洗的 `dangerouslySetInnerHTML`
- Auth Cookie：`httpOnly` + `secure` + `sameSite`
- 认证接口强制限流
- 生产环境强制 HTTPS + CORS + CSP 头

## 安装

**前置条件：** [Claude Code](https://claude.ai/code)（CLI 或 IDE 扩展）

### 方式一：克隆直接使用

```bash
git clone https://github.com/pplele/Multi-Agent-Skill.git
cd Multi-Agent-Skill
# 在此目录打开 Claude Code，Skill 自动从 .claude/skills/ 加载
```

```bash
mkdir -p 你的项目/.claude/skills
cp .claude/skills/dev-team.md 你的项目/.claude/skills/
```

```bash
mkdir -p ~/.claude/skills
cp .claude/skills/dev-team.md ~/.claude/skills/dev-team.md
```

无需重启，安装后立即可用。

## 确认命令速查

每个阶段转换都需要你输入明确的确认命令，系统不会自动推进。

| 转换 | 命令 |
|------|------|
| PM 完成 PRD → 技术评估 | `确认进入技术评估` |
| TechLead 完成设计 → 方案终审 | `确认进入方案终审` |
| 同意方案 → 开始编码 | `同意方案 v1.0` |
| 方案驳回 → 重来需求 | `不同意，返回状态1` |
| 方案驳回 → 重来设计 | `不同意，返回状态2` |
| 下一编码模块 | `继续下一模块` |
| 跳过某个模块 | `跳过模块 N` |
| 重新生成当前模块 | `重新生成当前模块` |
| QA 通过 → 交付 | `确认交付` |
| QA 发现问题 → 修复 | `要求修复` |
| 回到需求调整 | `调整需求` |
| 随时暂停 | `暂停` |
| 强制指定技术栈 | `强制使用 FastAPI` |

## 默认技术栈

当你不指定技术栈时，TechLead 推荐以下组合：

| 层级 | 推荐 | 理由 |
|------|------|------|
| 前端框架 | Next.js 14+ (App Router) | SSR/SSG，文件路由，Vercel 生态 |
| 语言 | TypeScript（strict 模式） | 全栈类型安全 |
| 样式 | Tailwind CSS 3.4+ | Design Token 映射为 Tailwind 配置 |
| 组件库 | shadcn/ui | 无障碍、可定制、复制即用 |
| 图标 | lucide-react | 风格统一的线性图标 |
| 后端框架 | FastAPI (Python) 或 Express (Node) | 性能 vs 生态的选择 |
| ORM | Prisma / SQLAlchemy 2.0 | 类型安全查询 + 迁移 |
| 数据库 | PostgreSQL 16 | JSON 支持、索引强劲 |
| 缓存 | Redis | 会话存储、限流、热数据 |
| 认证 | NextAuth.js / JWT + OAuth2 | 基于标准协议 |
| 部署 | Docker + docker-compose | 环境一致，易于扩展 |
| CI/CD | GitHub Actions | 公开仓库免费，配置简单 |

随时可以用 `强制使用 [技术名]` 覆盖，或在 TechLead 询问时指定完整技术栈。

## 常见问题

**Q: 能用于已有项目吗？加功能、改代码可以吗？**
可以。v4.2 起支持「已有项目修改模式」。在已有代码目录调用 `/dev-team` 时，TechLead 先做代码库侦查，输出项目基线报告，然后 PM 基于现有架构做增量需求澄清。从加一个字段到重构一个模块，都走完整流程。

**Q: 需要安装 Python、LangGraph 或其他运行时吗？**
不需要。零依赖。Claude Code 自己读取 Skill 定义并轮换角色，纯 Markdown 驱动。

**Q: 能自定义设计风格吗？**
可以。随时说"参考 Stripe 风格"、"用 #3b82f6 做主色"、"亮色模式"、"更活泼一点"。前端角色会自动适配。

**Q: 能强制指定技术栈吗？**
可以。TechLead 询问时回复 `强制使用 [你的技术栈]`。TechLead 会照做，但会标注潜在风险。

**Q: 不同意某个角色的输出怎么办？**
用驳回命令回退。`不同意，返回状态1` 回退到 PM 重新澄清需求。`不同意，返回状态2` 回退到 TechLead 重新设计。

**Q: QA 一直发现 Bug 怎么办？**
Bug 修复循环上限 3 轮。第 3 轮后如果还有阻断问题，系统强制暂停，给出三个选项：接受风险强制交付、继续修复、缩减需求范围。

**Q: 能跳过某些阶段吗？**
不能。工作流是严格线性的，不可跳阶段。但你可以在编码阶段用 `跳过模块 N` 跳过单个模块。

**Q: 能适配中文以外的语言吗？**
角色提示词是中文写的。你可以翻译 — 直接编辑 `.claude/skills/dev-team.md` 里的角色定义部分即可。
